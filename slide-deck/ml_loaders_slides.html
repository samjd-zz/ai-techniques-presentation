üí° Code Learning: Label Encoding

KEY CONCEPT

Why Label Encoding for Decision Trees?

Decision trees treat encoded values as categories, not mathematical relationships. They don't assume that 1 > 0 or that "northwest" (3) > "northeast" (0).

# Convert categories to integers for column in X.columns: if X[column].dtype == 'object': le = LabelEncoder() X_encoded[column] = le.fit_transform(X[column]) # Example: ['male', 'female'] ‚Üí [1, 0]

‚úì Works For

Decision Trees, Random Forests (treats as categories)

‚úó Don't Use For

Linear Regression, Neural Networks (implies false ordering)

Alternative: One-Hot Encoding

Creates binary columns: sex_male=[1,0], sex_female=[0,1]
Use for linear models and neural networks where ordering matters.

üå≤ Code Learning: Recursive Tree Traversal

ADVANCED PATTERN

Nested Function (Closure) Pattern

The recurse() function is defined inside extract_rules() to access outer variables without passing them as parameters.

def extract_rules(self, max_rules=10): rules = [] tree_structure = self.best_model.tree_ # NESTED FUNCTION - Closes over outer variables def recurse(node, depth, rule): if tree_structure.feature[node] == TREE_UNDEFINED: # LEAF NODE: Make prediction rules.append(rule) else: # INTERNAL NODE: Split left/right recurse(left_child, depth+1, left_rule) recurse(right_child, depth+1, right_rule) recurse(0, 0, "") # Start from root

Why Use Closures?

  • Cleaner signature: No need to pass rules, tree_structure every call
  • Encapsulation: recurse() only exists where needed
  • Standard pattern: Common for recursive algorithms with shared state

üéØ Code Learning: Boolean Masking

NUMPY OPTIMIZATION

Vectorized Operations: No Loops!

Boolean masking creates a True/False array, then uses it to filter rows in a single, highly optimized operation.

def get_features_by_activity(self, activity_name): activity_col = self.pd_df['Activity'].values # BOOLEAN MASK: Vectorized comparison (no loops!) mask = activity_col == activity_name # Result: [False, True, False, True, True, ...] # FANCY INDEXING: Two-step selection # [mask] ‚Üí Boolean indexing on rows # [:, self.feature_cols] ‚Üí Fancy indexing on columns return self.np_data[mask][:, self.feature_cols]

Performance Benefits

  • 1000x faster: C-level operations vs Python loops
  • Memory efficient: Single pass through data
  • Readable: Intent is clear and concise

Example: array1 = [12, 24, 16, 21, 32, 29, 7, 15]
mask = array1 > 20 ‚Üí [False, True, False, True, True, True, False, False]
array1[mask] ‚Üí [24, 21, 32, 29]

üìê Z-Score Normalization

z = (x - Œº) / œÉ

Where: x = original value, Œº = mean, œÉ = standard deviation

def normalize_features(self, data=None): data = self.pd_df.iloc[:, self.feature_cols].values.astype(float) mean = np.mean(data, axis=0) # Mean of each feature std = np.std(data, axis=0) # Std of each feature std[std == 0] = 1 # Avoid division by zero return (data - mean) / std # Standardize: mean=0, std=1

Why Normalize?

  • Equal scales: All features mean=0, std=1
  • Essential for KNN: Distance-based algorithms need equal scales
  • Prevents dominance: Large-scale features don't overshadow small ones
  • Improves convergence: Faster, more stable training

üîß Code Learning: Lambda Functions & Formatting

PYTHON TRICKS

FuncFormatter with Lambda: Format Y-Axis Ticks

Lambda functions are perfect for simple, one-time-use formatters without defining full functions.

# Format y-axis with comma separators (12000 ‚Üí 12,000) ax.yaxis.set_major_formatter( mtick.FuncFormatter(lambda x, _: f'{int(x):,}') ) # Lambda breakdown: # - x: the tick value (e.g., 12345.67) # - _: tick position (unused, underscore = convention) # - f'{int(x):,}': Format as integer with commas

Lambda vs Regular Function

# Lambda (concise) lambda x, _: f'{int(x):,}' # Equivalent function def format_tick(x, pos): return f'{int(x):,}'

Why Use Lambda?

  • One-line convenience
  • No need to name function
  • Perfect for callbacks
  • Keeps code localized

The Underscore Convention: In Python, _ means "I must accept this parameter but won't use it." Common in callbacks and unpacking.
